《鸟哥的Linux私房菜》正则表达式学习笔记

特殊符号及代表意义

特殊符号 代表意义
[:alnum:] 英文小写字符及数字，即 0-9, A-Z, a-z
[:alpha:] 英文大小写字符，即 A-Z, a-z
[:blank:] 空格键与tab键
[:cntrl:] 键盘上面的控制按键，亦即包括 CR, LF, Tab, Del.. 等等
[:digit:] 代表数字，亦即 0-9
[:graph:] 除了空白字符 (空格键 [Tab] 按键) 外的其他所有按键
[:lower:] 代表小写字符，亦即 a-z
[:print:] 代表任何可以被打印出來的字符
[:punct:] 代表标点符号 (punctuation symbol)，亦即：” ‘ ? ! ; : # $…
[:upper:] 代表大写字符，亦即 A-Z
[:space:] 任何会产生生空白的字符，包括空白鍵, [Tab], CR 等等
[:xdigit:] 代表 16 进制的数字类型，因此包括： 0-9, A-F, a-f 的数字与字符

grep进阶

命令:grep [-A] [-B] [--color=auto] '搜索字符串' filename
选项及参数:
    -A 后面可加数字,为after的意思,除了列出该行外,后面的n行也列出来
    -B 后面可加数字,为befer的意思,除了列出该列外,前面的n行业列出来
RE字符 意义 例子
^word 待搜素的字符串在行首 grep -n ‘^#’ regular_express.txt
word$ 待搜素的字符串在行尾 grep -n ‘!$’ regular_express.txt
. 一定有一个任意字符 grep -n ‘e.e’ regular_express.txt
|转义特殊符号 grep -n ' regular_express.txt
* 代表重复前一个字符0-无穷次 grep -n ‘ess*’ regular_express.txt
[list] 字符集合的RE字符,里面列出想要截取的字符 grep -n ‘g[ld]’ regular_express.txt
[n1-n2] 字符集合的RE字符,里面列出想要截取的字符范围 grep -n ‘[A-Z]’ regular_express.txt
[^list] 字符集合的RE字符,里面列出不要的字符串或范围 grep -n ‘oo[^t]’ regular_express.txt
{n,m} 连续n到m的前一个RE字符
若为{n}连续n个前一个RE字符
{n,}连续n个以上的前一个RE字符		 grep -n ‘go{2,3}g’ regular_express.txt

sed工具

命令:sed [-nefr] [动作]
选项及参数:
    -n 使用安静(silent)模式,在一般sed的用法中,所有来自stdin的资料一般都会列出到屏幕上,但如果加上-n参数后,则只有经过sed特殊处理的那一行(或者动作)才会被列出来
    -e 直接在命令模式上进行sed的动作编辑
    -f 直接将sed的工作写在一个文件内,-f filename 则可以执行filename内的sed动作
    -r sed的动作支持的是延伸型正规表达式语法(默认是基础语法)
    -i 直接修改读取的文件内容,而不是由屏幕输出
动作说明: [n1[,n2]]function
    n1,n2 不见得会存在,一般代表选择进行动作的行数,举例来说,如果我的动作是需要在10-20行之间进行的,则10,20(动作行为)

function下有这些东西:
a 新增,a的后面可以接字符串,而这些字符串会在新的一行出现(目前的下一行)
c 取代,c的后面可以接字符串,这些字符串可以取代n1,n2之间的行
d 删除,后面不用加任何东西
i 插入,i的后面可以接字符串,而这些字符串在新的一行出现(目前的上一行)
p 打印,将某个选择的资料打印
s 取代,可以直接进行替换的工作。可以搭配正则表达式,例如1,20s/old/new/g就是啦

延伸正则表达式

RE字符 意义 例子
+ 重复一个或一个以上的前一个RE字符 搜索god,good,goood等字符串
egrep -n ‘go+d’ regular_express.txt
? 0个或一个的前一个RE字符 搜索gd,god等字符串
egrep -n ‘go?d’ regular_express.txt
| 用or的方式找出字符串 egrep -n ‘gd|good’ regular_express.txt
() 找出群组字符 找出glad或good字符串
egrep -n ‘g(la|oo)d’ regular_express.txt
()+ 多个重复群组的判断 开头是A,结尾是C,中间有一个以上的xyz字符串的字符
echo ‘AxyzxyzxyzxyzC’,egrep ‘A(xyz)+C’

文件格式化及相关处理

格式化打印:printf

命令:printf '打印格式' 实际内容
选项及参数:
格式方面有以下几个特殊格式:
    \a 警告声音
    \b backspace键
    \f 清屏(form feed)
    \n 输出新的一行
    \r Enter按键
    \t 水平tab按键
    \v 垂直tab按键
    \xNN NN为两位数的数字,可以转换数字成为字符串
    关于C语言内,常见的变量格式
    %ns 那个n是数字,s代表string,即多少个字符
    %ni 那个n是数字,i代表integer,即多少整数位
    %N.nf 那个n与N都是数字,f代表floating(浮点),假设十个位数,小数点两位.%10.2f

awk:优良的文本处理工具

awk中,默认栏位的分割符是 空格键 或 tab键。

命令:awk '条件类型1{动作1} 条件类型2{动作2} ...' filename

awk内置变量

变量名称 意义
NF 每一行($0)拥有的栏位总数
NR 目前awk所处理的是第几行的资料
FS 目前得到分割字符,默认是空格键

awk的逻辑运算符

运算符号 意义
> 大于
< 小于
>= 大于或等于
<= 小于或等于
== 等于
!= 不等于

文件对比工具

diff

命令: diff [-bBi] from-file to-file
选项及参数:
    from-file
    to-file
    from-file和to-file可以用-替换,代表stdin的意思

    -b 忽略一行中,仅有多个空白的差异(例如"about me" 与 "about     me"视为相同)
    -B 忽略空白行差异
    -i 忽略大小写差异

cmp

命令:cmp [-l] file1 file2
选项及参数:
    -l 显示十进制的字节数和八进制的不同字节

patch

命令:patch -p0 < patch_file <==更新
    patch -R -pN < patch_file <==还原
选项及参数:
    -p 后面可以接取消几层目录的意思
    -R 代表还原,将新的档案还原成原来旧的版本
