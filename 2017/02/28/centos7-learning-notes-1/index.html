《鸟哥的Linux私房菜》文件权限与目录配置学习笔记

Linux文件属性详解

dr-xr-x--- 1 root root 103 Feb 27 23:00 .viminfo

第一栏:dr-xr-x—

第一个字符表示文件类型

  • d 目录

  • - 文件

    1. 纯文字文件
    2. 二进制文件
    3. 资料格式文件

  • l 软连接

  • b 硬件里面的可存储设备(block块设备),储存资料,提供系统读取的设备,比如硬盘,软盘

  • c 硬件里面的一次性设备,比如键盘、鼠标(character字符设备)

  • s 资料接口档、网络接口,经常在/run或/tmp中见到

  • p 资料传输文件 FIFO(first-in-first-out),pipe主要解决多个程序同时读取一个档案造成的错误问题

    接下来三个为一组,且都是[r(可读)w(可写)x(可执行)-(没有权限)]的组合

  • 第一组:文件所属者权限

  • 第二组:文件所属组权限

  • 第三组:其它人权限

第二栏:1

有多少文件链接到此节点

第三栏:root

文件所属者

第四栏:root

文件所属组

第五栏:103

文件大小,默认单位bytes

第六栏:Feb 28 23:00

文件最近修改日期,想要显示完整的时间格式 ls -l –full-time

第七栏:.viminfo

文件名,文件名前有.代表这个文件为隐藏文件 文件后缀与文件类型无关

  • Linux中,使用传统的Ext2/Ext3/Ext4文件系统以及xfs文件系统对文件名的长度限制为255bytes,大约128个汉字
    文件名命名时候最好避开以下特殊字符

      ? > < ; & ! [ ] | \ ' " ` ( ) { }

更改文件属性与权限

更改文件所属组chgrp

  • 命令:chgrp 组名 目录/文件名
  • 选项:
    -R 递归更改

更改文件所属者chown

  • 命令:chown 账号名称 目录或文件
    chown 账号名称:组名 目录或文件
  • 选项:
    -R 递归更改

更改文件权限chmod

使用数值类型变更文件权限

  • 命令:chmod xyz 目录或文件
    • xyz 权限属性数值的和
      • r:4
          w:2
          x:1
          rwx为7
          rw-为6
          r-x为5
          —为0
        例如想将权限改为[-rwxr-xr–],那么xyz为[4+2+1][4+0+1][4+0+0]=754
  • 选项:
    -R:递归更改

使用符号类型更改文件权限

chmodg(group群组)
o(others其它用户)
a(all全部身份)		+
-
=		r
w
x		文件或目录
  • 常见用法:
    chmod u=rwx,go=rx .bashrc
    chmod a+w .bashrc
    chmod a-x .bashrc

目录和文件的权限意义

权限对文件的意义

  • r:可读取文件内容
  • w:编辑、新增、修改该文件内容,但不具有删除该文件本身的权限
  • x:文件具有被系统执行的权限

权限对目录的意义

  • r:读取目录结构权限,可以查询该目录下文件名
  • w:改动该目录结构权限,包括
    • 新建文件或目录
    • 删除已经存在的文件或目录(与文件权限无关)
    • 将已存在的文件改名
    • 更改该目录内文件、目录位置
  • x:使用者是否能进入该目录
