《鸟哥的Linux私房菜》正则表达式学习笔记

特殊符号及代表意义

特殊符号 代表意义
[:alnum:] 英文小写字符及数字，即 0-9, A-Z, a-z
[:alpha:] 英文大小写字符，即 A-Z, a-z
[:blank:] 空格键与tab键
[:cntrl:] 键盘上面的控制按键，亦即包括 CR, LF, Tab, Del.. 等等
[:digit:] 代表数字，亦即 0-9
[:graph:] 除了空白字符 (空格键 [Tab] 按键) 外的其他所有按键
[:lower:] 代表小写字符，亦即 a-z
[:print:] 代表任何可以被打印出來的字符
[:punct:] 代表标点符号 (punctuation symbol)，亦即：” ‘ ? ! ; : # $…
[:upper:] 代表大写字符，亦即 A-Z
[:space:] 任何会产生生空白的字符，包括空白鍵, [Tab], CR 等等
[:xdigit:] 代表 16 进制的数字类型，因此包括： 0-9, A-F, a-f 的数字与字符

grep进阶

命令:grep [-A] [-B] [--color=auto] '搜索字符串' filename
选项及参数:
    -A 后面可加数字,为after的意思,除了列出该行外,后面的n行也列出来
    -B 后面可加数字,为befer的意思,除了列出该列外,前面的n行业列出来
RE字符 意义 例子
^word 待搜素的字符串在行首 grep -n ‘^#’ regular_express.txt
word$ 待搜素的字符串在行尾 grep -n ‘!$’ regular_express.txt
. 一定有一个任意字符 grep -n ‘e.e’ regular_express.txt
|转义特殊符号 grep -n ' regular_express.txt
* 代表重复前一个字符0-无穷次 grep -n ‘ess*’ regular_express.txt
[list] 字符集合的RE字符,里面列出想要截取的字符 grep -n ‘g[ld]’ regular_express.txt
[n1-n2] 字符集合的RE字符,里面列出想要截取的字符范围 grep -n ‘[A-Z]’ regular_express.txt
[^list] 字符集合的RE字符,里面列出不要的字符串或范围 grep -n ‘oo[^t]’ regular_express.txt
{n,m} 连续n到m的前一个RE字符
若为{n}连续n个前一个RE字符
{n,}连续n个以上的前一个RE字符		 grep -n ‘go{2,3}g’ regular_express.txt

sed工具

命令:sed [-nefr] [动作]
选项及参数:
    -n 使用安静(silent)模式,在一般sed的用法中,所有来自stdin的资料一般都会列出到屏幕上,但如果加上-n参数后,则只有经过sed特殊处理的那一行(或者动作)才会被列出来
    -e 直接在命令模式上进行sed的动作编辑
    -f 直接将sed的工作写在一个文件内,-f filename 则可以执行filename内的sed动作
    -r sed的动作支持的是延伸型正规表达式语法(默认是基础语法)
    -i 直接修改读取的文件内容,而不是由屏幕输出
动作说明: [n1[,n2]]function
    n1,n2 不见得会存在,一般代表选择进行动作的行数,举例来说,如果我的动作是需要在10-20行之间进行的,则10,20(动作行为)

function下有这些东西:
a 新增,a的后面可以接字符串,而这些字符串会在新的一行出现(目前的下一行)
c 取代,c的后面可以接字符串,这些字符串可以取代n1,n2之间的行
d 删除,后面不用加任何东西
i 插入,i的后面可以接字符串,而这些字符串在新的一行出现(目前的上一行)
p 打印,将某个选择的资料打印
s 取代,可以直接进行替换的工作。可以搭配正则表达式,例如1,20s/old/new/g就是啦

延伸正则表达式

RE字符 意义 例子
+ 重复一个或一个以上的前一个RE字符 搜索god,good,goood等字符串
egrep -n ‘go+d’ regular_express.txt
? 0个或一个的前一个RE字符 搜索gd,god等字符串
egrep -n ‘go?d’ regular_express.txt
| 用or的方式找出字符串 egrep -n ‘gd|good’ regular_express.txt
() 找出群组字符 找出glad或good字符串
egrep -n ‘g(la|oo)d’ regular_express.txt
()+ 多个重复群组的判断 开头是A,结尾是C,中间有一个以上的xyz字符串的字符
echo ‘AxyzxyzxyzxyzC’,egrep ‘A(xyz)+C’

查询命令是否为bash shell的内置命令:type

命令:type type [-tpa] name
选项及参数: 不加任何选项和任何参数,type会显示出name是外部命令还是内置命令
    -t type将会以以下的字段显示出他的意义

变量的输入

echo $variable 或者 echo ${variable}
变量的设置规则:

  1. 变量与变量内容以一个等号连接
    myname=haokaiyang
  2. 等号两边不能直接跟空格,错误示范
    myname = haokaiyang 或者myname=hao kai yang
  3. 变量名称只能是英文字母与数字,不能以数字开头
  4. 变量内容有空白字符可以使用单引号或双引号将变量内容结合起来
    myname=“hao kai yang”
  5. 可以用跳脱字符\将特殊字符[enter],$,,空格,’变成一般字符,如
    myname=hao\ kai\ yang
  6. 在一串命令的执行中,还需要用到其它命令时候,可以使用反引号指令或$(指令)
  7. 若该变量为扩增内容时候,则可用”$变量名称”或${变量}累加内容
    PATH=”$PATH”:/home/bin或PATH=${PATH}:/home/bin
  8. 若该变量需要在子程序执行,需要export来使变量变成环境变量
    export PATH
  9. 通常大写字符为系统变量,小写为自定义变量
  10. unset取消设置变量 unset 变量名称

与文件系统和程序的限制关系 ulimit

为了限制使用者占用系统资源,包括可以开启的文件数量,可以使用的CPU时间,可以使用的内存总量,可以用ulimit设置

命令:ulimit [-SHacdfltu] [配额]
选项及参数:
    -H hard limit 严格设置,不能超过这个设置的述职
    -S soft limit 警告设置,可以超过这个值,但是会警告
    在设置上,通常soft比hard小
    -a 后面不接任何选项及参数,可以列出所有的限制额度
    -c 当某些程序发生错误时,系统可能会将该程序在内存中的资讯写成文件(     除错用),这种档案就被称为核心文件(core file),此为限制每个核心档案的最大容量
    -f 此shell可以建立的最大文件容量(一般可能设置为2GB),单位为Kbytes
    -d 程序可以使用的最大分段容量
    -l 可以用于锁定(lock)的内存容量
    -t 可以使用的最大CPU时间(单位为秒)
    -u 单一使用者可以使用的最大程序(process数量)

变量设置方式 说明
${variable#关键字} 若变量内容从头开始的资料符合[关键字],则将符合的最短资料删除
${variable##关键字} 若变量内容从头开始的资料符合[关键字],则将符合的最长资料删除
${variable%关键字} 若变量内容从尾向前的资料符合[关键字],则将符合的最短资料删除
${variable%%关键字} 若变量内容从尾向前的资料符合[关键字],则将符合的最长资料删除
${variable/旧字符串/新字符串} 若变量内容符合旧字符串,则第一个旧字符串会被新字符串替换
${variable//旧字符串/新字符串} 若变量内容符合旧字符串,则全部的旧字符串会被新字符串取代

login与non-login shell

login shell:取得bash时候需要完整的登录流程,就称为login shell,举例来说,你要由tt1-tty6登录,需要输入使用者的账号与密码,此时取得的bash就称为 login shell
non-login shell:取得bash界面的方法不需要重复登录,举例来说：用窗口模式登录Linux后,再启动终端,不需要输入账号密码,那个bash的环境称为 non-login shell,在原本的bash环境中再次下达bash命令,同样也没有输入账号密码,子程序也是non-login-shell
login shell 只会读 
/etc/profile 系统整体的设定,最好不要修改
/.bash_profile或/.bash_login或~/.profile:属于个人资料
/etc/profile (login shell才会读)
每个使用者登录使用bash一定会读取的配置文件
配置文件主要变量
PATH：根据UID决定PATH要不要含有sbin的命令指令
MAIL：根据账号设置好使用者的mailbox到/var/spool/mail/账号名称
USER: 根据使用者的账号设置此变量内容
HOSTNAME:根据主机的hostname命令决定这个变量
HISTSIZE：历史命令记录笔数,CentOS7默认1000
umask包括root默认为022,而一般用户为002
这个文件还会引入其他文件,在CentOS7默认情况下,会引入下列文件
/etc/profile.d/*.sh
/etc/locale.conf 是由/etc/profile.d/lang.sh呼叫进来的
/usr/share/bash-completion/completions/*是由/etc/profile.d/bash_completion.sh这个文件载入的
~/.bash_profile(login shell才会读)
bash在读完了整个环境设定的/etc/profile并呼叫其他设定档后,在login shell的bash环境中,所读取的个人偏好设置其实主要有三个.顺序如下

  1. ~/.bash_profile
  2. ~/.bash_login
  3. ~/.profile

login shell 只会读以上三个文件的其中一个,读取顺序如上,如果~/.bash_profile存在,那后面的都不会引入

    -v 默认不启动,若启用后,在信息被输出前,会先显示信息的原始内容
    -x 默认不启动,若启用后,在指令被执行前,会显示命令内容(前面有++符号)
    -h 默认启动。与历史命令有关
    -H 默认启动。与历史命令有关
    -m 默认启动。与历史命令有关
    -B 默认启动。与括号[]的作用有关
    -C 默认不启动,若使用>等,则若档案存在时,该文件不会被重置

命令执行判断依据

cmd;cmd(不考虑相关性连接命令)

$?与&& ||

cmd1&&cmd2 1.若cmd执行完毕且正确执行($?=0),则开始执行cmd2
           2.若cmd执行完且为错误($?=0),则cmd2不执行
cmd1||cmd2 1.若cmd执行完毕且正确执行($?=0),则cmd2不执行
           2.若cmd执行完且为错误($?=0),则开始执行cmd2

截取命令cut,grep

命令:cut -d '分割字符' f fields
    cut -c 字符区间
选项及参数:
    -d:后面接分割字符,与-f一起使用
    -f:依据-d分割字符将一段信息成为字段,取-f取出第几段的意思
    -c:以字符(characters)的单位取出固定字符区间
命令:grep [-acinv] [--color=auto] '搜索字符串' filename
选项及参数:
    -a 将binary文件以text文件的方式搜索信息
    -c 统计找到'搜索字符串'的次数
    -i 忽略大小写的不同,所以大小写视为相同
    -n 输出行数
    -v 反向选择,表示显示出没有'搜索字符串'内容的那一行
    --color=auto 可以将找到的关键字部分加上颜色的显示

参数替换:xargs

由于很多命令不支持管道,所以我们通过xargs来引用stdin

命令:xargs [-0epn] command
选项及参数:
    -0 将stdin中的字符还原为特殊字符
    -e 这个是EOF(end of file)的意思,后面接一个字符串,当xargs分析到这个字符,就会停止工作
    -p 在执行每个命令的argument时,询问使用者的意思
    -n 后面接次数,每个command命令执行时,要使用几个参数的意思
    当xargs后面没有接任何的命令时候,默认是以echo来进行输出
